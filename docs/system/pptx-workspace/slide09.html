WHAT IT LOOKS LIKE

A typical session, from setup to commit

Four commands, four artifacts, one green CI run:

$ /setup docs/research/my-project-notes.md
→ configures PROJECT.md, CLAUDE.md, and initial docs
→ installs stack scaffolding if specified

$ /explore add a health check endpoint
→ asks 3 clarifying questions
→ creates docs/features/FEAT-001_health-check/prd.md

$ /build FEAT-001
→ RED: tdd-test-writer writes failing tests (sees PRD only)
→ GREEN: tdd-implementer writes minimal code (sees tests only)
→ REFACTOR + QA: clean up → 15 SAST rules + OWASP → APPROVED

$ /commit
→ QA gate passes → conventional commit → CI passes in 12s
